# 现代黑暗生存文字冒险游戏 - 开发进度追踪

本文档旨在拆解并追踪《现代黑暗生存文字冒险》Web端游戏的开发进度。游戏整体基调为黑暗、压抑、悬疑，以剧情为核心，包含丰富的NPC交互、网状任务路线以及多线叙事。

## 阶段一：项目初始化与底层架构搭建 (Project Setup & Core Framework)
- [x] **1.1 技术栈选型与项目初始化**：使用 Vue3 + TypeScript 作为核心技术栈，初始化项目结构，配置代码规范（ESLint, Prettier）。
- [ ] **1.2 模块化架构设计**：规划项目目录结构（组件、状态管理、工具类、剧情脚本库等），禁止单文件堆砌代码。
- [ ] **1.3 i18n 多语言系统**：搭建国际化系统，初步配置简体中文环境。
- [ ] **1.4 核心状态管理**：搭建全局游戏状态管理器（如 Redux, Zustand 或 Pinia），管理玩家属性、时间、背包、NPC好感度、世界状态等。

## 阶段二：UI/UX 视觉与交互基础 (UI/UX Foundation)
> 设计原则：深色主题、专业图标（禁用Emoji）、拒绝“大字报”排版、禁用原生alert/confirm、现代化简约设计、藏色与层次感、MacOS/iOS级动画体验。
- [ ] **2.1 全局主题与基础组件**：定义全局CSS变量（深色系色板、排版规范），封装基础按钮、卡片、输入框。
- [ ] **2.2 自定义弹窗系统 (Dialog System)**：实现一套高度可定制的 Dialog 组件，完全替代原生的 alert, confirm, prompt，并带有平滑的出现/消失动画。
- [ ] **2.3 核心界面布局 (Layout)**：设计并实现带有层次感的 Header、定制化导航栏。
- [ ] **2.4 游戏功能面板开发**：
  - [ ] **网状地图面板**：实现节点式的行动区域地图。
  - [ ] **日志/剧情面板**：呈现当前发生的事件文字，具有良好的阅读排版。
  - [ ] **状态栏与能力面板**：展示玩家的属性、生理/心理状态。
  - [ ] **背包系统面板**：管理获得的道具、线索。
  - [ ] **行动/选项面板**：动态渲染玩家当前可执行的操作。

## 阶段三：核心游戏机制 (Core Game Systems)
- [ ] **3.1 时间管理系统**：实现“时间流逝”机制，玩家的所有探索、交互行为都会消耗时间。统筹规划总计90天的日程表。
- [ ] **3.2 日程机制 (Rest Day vs Game Day)**：
  - [ ] 实现“休息日”逻辑：开放广域探索，允许接取NPC支线，培养属性与关系。
  - [ ] 实现“游戏日”逻辑：锁定区域，强制推进主线残酷游戏，根据前置条件（线索、能力、关系）动态生成应对选项。
- [ ] **3.3 角色创建系统**：实现开局自定义系统。允许玩家设定性别、性格特质、过往背景等（这些设定需在全局状态中注册，并能在后续剧情中作为解锁关键选项的 key）。
- [ ] **3.4 天气系统**：设计常规天气与特殊天气（如暴雨、血雾等），天气改变将影响可探索区域或触发隐秘事件。
- [ ] **3.5 简易战斗系统**：实现服务于剧情的战斗判定机制（非重度策略，偏向数值检定与关键道具使用）。

## 阶段四：庞大NPC与叙事系统支持 (Narrative & NPC Systems)
- [ ] **4.1 关系网与好感度系统**：定义每个NPC的独立好感度/信任度/敌恶度数值，支持两面三刀、背叛等复杂关系逻辑。
- [ ] **4.2 剧本解析引擎**：实现一个能读取 JSON 或 Markdown 格式剧本的解析器，支持根据条件（如“敏捷>5”、“拥有线索A”、“性格=冷酷”）渲染不同的对话分支。
- [ ] **4.3 任务树与事件分发机制**：实现一套网状任务追踪系统，支持主线事件强制触发、支线事件条件触发、天气限定事件触发。

## 阶段五：剧情编排与内容填充 (Content Creation)
> 编剧要求：使用 Mermaid 绘制流程图；剧情黑暗压抑但不失人性微光；适时加入黑色幽默或放松情节；文字有深度但易懂。
- [ ] **5.1 剧本大纲与架构文档**：创建 `PLOT_OUTLINE.md`，使用 Mermaid 规划90天的主线节点与分支走向。
- [ ] **5.2 序章开发 (Awakening)**：角色创建、密室苏醒、首次“残酷游戏”体验、神秘组织初露端倪。
- [ ] **5.3 第一阶段内容 (Day 1 - Day 30)**：
  - 建立初始的人际关系网，展现部分NPC的善与恶。
  - 设计至少3场核心“游戏日”关卡。
  - 规划第一个必定触发特殊天气的剧情关键点。
- [ ] **5.4 第二阶段内容 (Day 31 - Day 60)**：
  - 剧情反转与中期高潮：信任的崩塌与意想不到的盟友。
  - 逐渐揭露神秘组织的局部真相。
- [ ] **5.5 第三阶段内容 (Day 61 - Day 90)**：
  - 终局抉择：根据玩家90天积累的关系、线索和属性，走向不同的逃脱路线或毁灭结局。
  - 编写多结局文本。

## 阶段六：平衡性调整与测试打磨 (Polishing & Testing)
- [ ] **6.1 数值平衡**：调整每次行动消耗的时间、属性成长的难度、“游戏日”判定所需的数值门槛。
- [ ] **6.2 视觉与动效打磨**：全面复查UI细节，确保色彩层次感、转场动画符合 MacOS/iOS 级别的顺滑体验。
- [ ] **6.3 剧本润色与逻辑自洽**：通读所有支线和主线，确保多线叙事下没有逻辑死角或“看不懂”的谜语人台词。
- [ ] **6.4 多周目体验优化**：确保初始创建角色的不同设定，能够实质性地改变游戏体验，鼓励重复游玩。
