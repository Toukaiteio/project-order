# 现代黑暗生存文字冒险游戏 - 开发进度追踪

本文档旨在拆解并追踪《现代黑暗生存文字冒险》Web端游戏的开发进度。游戏整体基调为黑暗、压抑、悬疑，以剧情为核心，包含丰富的NPC交互、网状任务路线以及多线叙事。

## 阶段一：项目初始化与底层架构搭建 (Project Setup & Core Framework) - [DONE]
- [x] **1.1 技术栈选型与项目初始化**：使用 Vue3 + TypeScript 作为核心技术栈，初始化项目结构，配置代码规范（ESLint, Prettier）。
- [x] **1.2 模块化架构设计**：规划项目目录结构（组件、状态管理、工具类、剧情脚本库等），禁止单文件堆砌代码。
- [x] **1.3 i18n 多语言系统**：搭建国际化系统，初步配置简体中文环境。
- [x] **1.4 核心状态管理**：搭建全局游戏状态管理器（Pinia），管理玩家属性、时间、背包、NPC好感度、世界状态等。

## 阶段二：UI/UX 视觉与交互基础 (UI/UX Foundation) - [DONE]
> 已实现：深色主题、网状地图 (Canvas)、剧情日志流 (Story Log)、状态侧边栏、背包系统、iOS 风格动效。
- [x] **2.1 全局主题与基础组件**：定义全局CSS变量（深色系色板、排版规范），封装基础按钮、卡片、输入框。
- [x] **2.2 自定义弹窗系统 (Dialog System)**：实现一套高度可定制的 Dialog 组件，完全替代原生的 alert, confirm, prompt，并带有平滑的出现/消失动画。
- [x] **2.3 核心界面布局 (Layout)**：设计并实现具有层次感的 Header、定制化导航栏、网状地图带。
- [x] **2.4 游戏功能面板开发**：
  - [x] **网状地图面板**：基于 Canvas 实现了节点式的行动区域地图。
  - [x] **日志/剧情面板**：实现了 Narrative Entry 与 Action Choice 混排的流式交互。
  - [x] **状态栏与能力面板**：侧边栏展示玩家的生理/心理状态及能力点。
  - [x] **背包系统面板**：实现了带有详情预览的列表式背包。

## 阶段三：核心叙事引擎与角色系统 (Narrative Engine & Character System) - [DONE]
- [x] **3.1 角色创建系统**：实现了具有沉浸感序幕的开局自定义系统，支持姓名、性别、特质与属性点分配。
- [x] **3.2 数据驱动的行动系统**：开发了 `usePlot` 引擎，支持基于 JSON/TS 脚本的剧情跳转、前置条件判定与逻辑回调。
- [x] **3.3 物品与线索交互逻辑**：实现了剧情效果对背包系统的修改，支持在剧情中获取道具和文档。
- [x] **3.4 全局自定义对话框**：封装了 `GameDialog` 组件与 `dialogStore`，支持符合整体 UI 风格的 Alert 和 Confirm。

## 阶段四：庞大NPC与日程管理 (NPCs & Schedule) - [DONE]
- [x] **4.1 NPC 深度交互模型**：实现了 NPC 独立好感度系统，及基于位置的动态偶遇逻辑。
- [x] **4.2 90天日程调度引擎**：实现了 `ScheduleStore` 与跨天自动切换逻辑，支持天气随机化与主线触发。

- [x] **4.4 地图节点移动与解锁**：完善了地图节点的动态移动逻辑，并集成了随机偶遇判定。
